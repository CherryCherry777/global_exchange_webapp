7.2 Configurar terminales
=========================

Requisitos previos
------------------

- Tener identificador y credenciales del terminal (`tauser_id`, `api_key`).
- Conexión de red estable y permisos para comunicar con los endpoints del backend.

Pasos de configuración
----------------------

1. En el panel de administración, crear o registrar un nuevo terminal con `tauser_id` y metadatos (ubicación, responsable).
2. Proveer credenciales seguras al dispositivo (API key o certificado) y configurar URL de backend.
3. Verificar conectividad y sincronización de hora para evitar desajustes en registros.

Opciones avanzadas
------------------

- Configurar modos offline: almacenar transacciones localmente y sincronizar cuando el terminal recupere conectividad.
- Ajustes de denominaciones y caja inicial por terminal.

**11.1 Términos técnicos**
==========================

Cotización
----------
Precio al que se compra o vende una moneda en un momento dado. En el proyecto suele distinguirse entre `buy_rate` y `sell_rate`.

Moneda / Currency
-----------------
Unidad de intercambio (por ejemplo, USD, EUR, PYG). Las monedas tienen campos como `decimales` y formatos de presentación.

Billetera
---------
Entidad que representa fondos digitales asociados a un cliente o negocio. Puede tener varias denominaciones y métodos de acceso.

Transacción
-----------
Registro de una operación de intercambio, compra/venta o movimiento de fondos: incluye montos, comisiones, origen y destino.

Límite de intercambio
---------------------
Restricción configurada por categoría o cliente que limita el monto máximo que puede intercambiarse en un periodo.

Tauser
------
Objeto/entidad usada por el sistema para representar terminales o agentes que realizan operaciones (nombre usado en este proyecto).

Autenticación / Autorización
----------------------------
Autenticación: comprobar la identidad (login). Autorización: verificar permisos para realizar acciones.

Runbook
-------
Documento que describe pasos a seguir ante incidentes (por ejemplo, pasos para recuperación y contacto de emergencia).

Webhook
-------
Endpoint HTTP que recibe notificaciones de eventos desde otro sistema (por ejemplo, notificación de pago).

API
---
Interfaz de programación (endpoints HTTP/REST) que permite interactuar con el sistema desde clientes externos.
